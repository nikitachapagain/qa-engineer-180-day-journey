# 🚀 Day 6 – Introduction to Manual Testing

## 📚 What I Learned Today

Today, I started learning **Manual Testing**, which is the foundation of Software Quality Assurance (QA). Manual testing involves verifying software functionality without using automation tools. Testers execute test cases manually, compare actual results with expected results, and identify defects before the software is released.

## 📖 Topics Covered

* Introduction to Manual Testing
* What is Software Testing?
* Importance of Manual Testing
* Roles and Responsibilities of a Manual Tester
* Advantages and Limitations of Manual Testing
* Manual Testing vs. Automation Testing
* Software Testing Principles

## 💡 Key Takeaways

* Manual testing helps ensure software quality from the end user's perspective.
* It is effective for exploratory, usability, and ad-hoc testing.
* Finding defects early reduces development costs and improves product reliability.
* A solid understanding of manual testing is essential before learning automation testing.
