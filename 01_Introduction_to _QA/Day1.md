# Day 1 - Introduction to QA (Software Testing Basics)

## 1. What is Software Testing?

Software testing is the process of evaluating and verifying that a software application works as expected, performs reliably, and meets its specified business and user requirements. Its core purpose is to identify defects, bugs, and gaps before the product reaches real users, minimizing the risk of system failure in production environments.
In simple words:
👉 We check if the software has any mistakes (bugs).

### Example:
A calculator app should give:
2 + 8 = 10

If it gives wrong output, it is a bug.

---

## 2. Why Software Testing is Important?

Software testing is important because:

- It finds bugs before users find them
- It improves software quality
- It saves company money
- It increases customer satisfaction
- It ensures software works properly

### Real-life example:
If a banking app shows wrong balance, users can lose money.
Testing prevents this.

---

## 3. What is QA (Quality Assurance)?

QA means making sure the process of building software is correct.

👉 QA focuses on preventing bugs.

### Example:
- Following proper steps while building software
- Writing correct requirements
- Planning testing properly

---

## 4. What is QC (Quality Control)?

QC means checking the final product to find bugs.

👉 QC focuses on finding bugs.

### Example:
- Testing login page
- Checking buttons work
- Finding errors in app

---

## 5. QA vs QC Difference

| QA (Quality Assurance) | QC (Quality Control) |
|------------------------|----------------------|
| Prevents bugs | Finds bugs |
| Focus on process | Focus on product |
| Done before development | Done after development |
| Improves quality | Checks quality |

Simple line:
👉 QA = Prevent  
👉 QC = Detect

---

## 6. What is SDLC (Software Development Life Cycle)?

SDLC is the step-by-step process of building software.

### Stages:

1. Planning
2. Requirement Analysis
3. Design
4. Development
5. Testing
6. Deployment
7. Maintenance

### Example:
Building a house:
- Plan house
- Draw design
- Build house
- Test safety
- Move in

---

## 7. What does a QA Engineer do?

A QA Engineer:

- Reads requirements
- Writes test cases
- Tests software
- Finds bugs
- Reports bugs
- Retests fixes
